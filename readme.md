# Teacher Tati — English Learning Platform Powered by AI

A full-stack English language learning platform built with FastAPI and vanilla JavaScript, featuring AI-powered conversation practice, gamification, administrative tools, and a subscription payment system.

---

## What it does

Teacher Tati is a conversational English learning app where students chat with an AI teacher named Tati. The AI responds in English, corrects grammar in real time, adapts to the student's proficiency level, and generates study materials on demand.

The platform goes beyond a simple chatbot — it includes a quiz and flashcard system, real-life conversation simulations (airport, restaurant, job interview), a streak-based engagement tracker, a monthly ranking, and a trophy collection system. Teachers have a full administrative dashboard with per-student insights generated by AI.

The project is live and used by real students. Payments are processed through Asaas (Brazilian payment gateway) with PIX, boleto, and credit card support.

## Architecture

```
Frontend (Vanilla JS + HTML/CSS)    →    Vercel
Backend (FastAPI / Python)           →    Railway
Database                             →    Supabase (PostgreSQL)
Cache & Rate Limiting               →    Upstash Redis
LLM Providers                       →    Groq (primary), Claude, Gemini (fallback)
Text-to-Speech                      →    Edge TTS (free), gTTS (fallback)
Payments                            →    Asaas
Error Monitoring                    →    Sentry
Email                               →    Resend API
```

The backend uses a multi-key Groq rotation system — up to 5 API keys cycle automatically on rate limit or auth errors, keeping the service available even under heavy use.

## Tech Stack

**Backend**
- Python 3.12, FastAPI, Pydantic v2
- Supabase Python client (PostgreSQL via service key)
- python-jose for JWT, bcrypt for password hashing
- Groq SDK, Anthropic SDK, Google GenAI
- edge-tts and gTTS for audio synthesis
- httpx for async HTTP (Asaas integration)
- Sentry FastAPI integration
- Upstash Redis for per-route rate limiting

**Frontend**
- No framework — plain HTML, CSS with custom properties, vanilla JavaScript
- WebSocket client for real-time streaming chat
- i18n system built from scratch (pt-BR, en-US, en-UK)
- Toastify for notifications, Marked.js for markdown rendering, DOMPurify for sanitization
- Font Awesome 6, Google Fonts (Sora + DM Sans)

**Infrastructure**
- Railway (backend, auto-deploy from GitHub)
- Vercel (frontend, static hosting)
- Supabase (database + storage for module content)

## Features by Area

### Authentication
- Email/password login with bcrypt hashing
- Google OAuth via Google Identity Services
- Password recovery with temporary password sent via Resend
- JWT with configurable expiry (default 15 days)

### Chat
- Real-time streaming via WebSocket
- AI adapts response length and vocabulary to the student's level (Beginner to Advanced)
- Per-conversation history stored in Supabase
- File attachment support: PDF, DOCX, and plain text are parsed and sent as context
- Audio recording → Whisper Large V3 transcription → text sent to LLM
- Text-to-Speech on every AI response
- Word tooltip: click any English word in a message to see translation, phonetics, and pronunciation audio
- Students can generate a PDF study report from any conversation

### Activities
- **Quizzes**: Multiple-choice, 4 options, 3 attempts per quiz. Questions can be manually created or AI-generated from module content. Results trigger trophy checks.
- **Flashcards**: Vocabulary packs with word, translation, and example sentence. Packs are AI-generated from a theme prompt.
- **Writing exercises**: Free-text submissions reviewed by teachers or auto-corrected by AI with a 0–100 score.
- **Simulations**: Roleplay scenarios stored in the database. Each scenario has a system prompt that defines the AI character. Students can use text or voice input.

### Gamification
- **Streaks**: A study day counts after 3+ messages in a single day. Current and longest streak tracked per user.
- **Trophies**: ~30 trophies across categories (quizzes, streak, messages, milestones). Checked and awarded automatically after relevant actions.
- **Monthly ranking**: Score calculated from study sessions (message = 8 pts, quiz = 7 pts, simulation = 10 pts, exercise = 5 pts, flashcard = 3 pts). Top 15 displayed with a countdown to end of month.
- **XP system**: Separate from ranking, maps to CEFR levels A1–C2.

### Admin Dashboard
- Overview stats, weekly activity bar chart, level distribution donut chart
- Student list with last active timestamp and message count
- Per-student AI insight (pedagogical report from conversation history)
- Per-student grammar error mapping
- Per-student interest analysis and study recommendations
- Module and flashcard CRUD with AI generation
- Submission correction queue with AI auto-correction
- Simulation scenario management

### Payments
- Asaas integration (sandbox and production)
- Recurring monthly subscriptions (Basic and Full plans, prices from database)
- PIX QR code generation on subscribe
- Webhook handler for PAYMENT_CONFIRMED, PAYMENT_OVERDUE, PAYMENT_DELETED events
- Grace period logic: subscription stays active until 5th business day of the following month after expiry
- Free tier: 5 messages before paywall

## Running Locally

**Requirements**: Python 3.12, a Supabase project, a Groq API key.

```bash
# Clone the repo
git clone https://github.com/your-org/teacher-tati
cd teacher-tati/backend

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Fill in: SUPABASE_URL, SUPABASE_KEY, JWT_SECRET_KEY, GROQ_API_KEY_1

# Run migrations in Supabase SQL editor (see backend/migrations/)

# Start the API
uvicorn main:app --reload --port 8000
```

```bash
# Serve the frontend (separate terminal)
cd frontend
python -m http.server 8080
```

Open `http://localhost:8080` and register an account.

For full functionality (TTS, payments, email), populate the remaining `.env` keys. The app degrades gracefully — missing keys disable the relevant feature rather than crashing.

## Database

The schema lives in `backend/migrations/`. Run them in order in the Supabase SQL editor. The main tables are:

- `users` — students and teachers with JSONB columns for streak data, vocabulary, XP, and study goals
- `conversations` and `messages` — chat history
- `modules`, `module_contents`, `quizzes`, `quiz_questions` — course content
- `user_progress` — quiz attempt records
- `trophies` and `user_trophies` — trophy definitions and awards
- `study_sessions` — time tracking for ranking calculation
- `subscriptions` — payment and access control
- `simulations` — roleplay scenario definitions

## Known Limitations

- The RAG system (ChromaDB + HuggingFace embeddings) is present in the codebase but disabled in production due to Railway build constraints. The `rag_search.py` service returns empty context silently.
- The animated avatar lip-sync uses Web Audio API volume analysis rather than a proper viseme pipeline. It works well enough but is approximate.
- The frontend has no build step — all JavaScript is loaded as separate files. For large teams, bundling would be a worthwhile investment.
